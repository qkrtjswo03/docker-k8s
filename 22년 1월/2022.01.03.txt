통신(네트워크)를 위해서는 아래와 같은 두가지 주소가 필요하다.
해당 주소는 NIC(Network Interface Card; LAN  카드)에 할당된다.
1. 물리적 주소 (MAC 주소)
   - 바꿀 수 없다
   - 사람의 신체(동일한 경우는 없다)
   - MAC 주소의 앞 절반 부분은 OUI 라고 하여 해당 NIC 을 제조한 
     회사에 부여되는 고유한 코드이다. 나머지 뒷 부분은 해당 제조회사에서
     랜덤하게(겹치지 않게) 부여한다. 

2. 논리적 주소 (IP 주소)
   - 그때 그때 필요할 때마다 바꿀 수 있다.  
   - 사람의 이름(성+이름)
   - 겹치는 경우가 나올 수 있다. 
     (동일 네트워크 내에서는 겹치지 않지만, 서로 다른 네트워크에서는 
      동일한 주소가 나올 수 있다)

IP 주소란?? (Internet Protocol)
- Internet (Internetworking) : 상호통신
- Protocol : 통신을 위한 규약(통신 언어)
상호통신을 위한 통신 규약(언어)
- IP는 표준 통신 규약(언어) 이므로 통신을 위한 모든 장비,장치에는 기본적으로 IP 가 할당되어야 한다. 
- 구분
  IPv4  vs. IPv6

일반적으로 IP 라고 하면 IPv4 를 의미한다. 
IP 구성
 - 32 비트
 - 4개의 옥텟으로 되어 있다. 
 - 각 옥텟은 8 비트로 표현한다. 
 - 0.0.0.0 ~ 255.255.255.255 까지 사용할 수 있다. 
 - IP 는 사람의 이름과 같은 역할 (홍길동)
   -> IP 주소의 일정 부분은 네트워크 영역(집안이름), 나머지 일정 영역은 호스트(집안에서 사용하는 이름) 영역이다. 

 IP 주소의 네트워크 영역과 호스트 영역을 구분하기 위하여 Subnet Mask 를 사용한다.
IP -> 1.1.1.1            -> 0000 0001.0000 0001.0000 0001. 0000 0001
SM -> 255.255.255.0 -> 1111 1111.1111 1111.1111 1111. 0000 0000
                                (고정 ----------------------------->)(마음대로!!)
 subnet mask 의 연속된 1 에 해당하는 자리가 네트워크 영역이다.
 subnet mask 의 연속된 0 에 해당하는 자리가 호스트 영역이다.
 
결국 1.1.1.1 255.255.255.0 의 네트워크 영역은 1.1.1 이 된다. 

 1.1.1.0 ~ 1.1.1.255 가 전체 범위가 된다.

DNS (Domain Name System) 
- 시나리오  
192.168.1.100(PC) 이 주소창에 www.snu.or.kr 을 입력했다면??
1. PC 에서 설정한 DNS 서버에게 query 한다.
   (www.snu.or.kr 의 IP 주소 좀 알려줘!!)
2. DNS 서버는 자신에게 등록된 도메인:IP 매핑 데이터를 검색하여 원하는 결과를 찾는다. www.snu.or.kr 의 주소가 만약 5.5.5.5 라면 해당 정보를 PC에게 알려준다.
3. PC 는 DNS 서버로 부터 응답을 받는다. 응답내용에는  
   snu.or.kr 은 5.5.5.5!
4. 최종적으로 PC는 5.5.5.5 로 웹 접속한다.!!

만약 PC에 DNS 서버의 주소를 작성하지 않았다면? 현실적으로 인터넷으로의 접속은 불가능하게 된다. 실제로 8.8.8.8 은 구글에서 운영하는 DNS 서버이다.



공인IP 와 사설 IP 
- 비용을 지불하느냐? 그렇지 않느냐?
공인 IP 는 ISP(Internet Service Provider-KT, LG U+, SK BB)에 비용을 지불하고 인터넷을 사용할 수 있는 공인 주소를 부여받는다.
사설 IP 는 별도의 비용지불이 없으므로 마음대로 써도 되지만 인터넷 사용/접근은 불가하다. 


Internet----ISP--->공인IP-->공유기----노트북(사설IP)
                                         |
                                       TV(사설IP)

위의 경우 사설IP 가 할당된 노트북과 TV는 인터넷을 사용할 수없다. 하지만 인터넷으로 나갈때 공유기는 해당 사설 주소를 "공인IP" 주소로 변경하여 내보낸다. 이를 통해 인터넷 연결이 가능해 진다. 우리는 이러한 기술을 Network Address Translation(NAT) 라 한다. 

 정리 :
 - 통신을 위해서는 IP가 필요하다.
 - 네트워크 범위를 알기 위해 서브넷 마스크가 필요하다.
 - 다른 네트워크와 통신을 위해서는 "기본 게이트웨이" 가 필요하다.
 - IP 를 이용하는 방법이 아닌 이름(도메인; www.naver.com)을 이용하여 
   통신을 하고자 한다면 "DNS 서버" 가 필요하다.
 - 인터넷을 사용하고 싶다면 "공인IP"가 있어야 한다.


- 공인 IP,, 사설 IP
- IP 를 구성하는 방법
  1) 정적 : 관리자 또는 사용자가 필요한 주소를 직접 입력하는 방식으로 변경될 소지가 없으므로 주로 서버에 적용한다.
  2) 동적(DHCP) : DHCP((Dynamic  Host  Configuration  Protocol)) 을 서버와 클라이언트가 주고 받으며 서버로 부터 필요한 IP,, Subnet  Mask,, Default  Gateway,, DNS  등의 정보를 할당 받아 사용하는 방식. 주로 PC 에 적용시키는 방법.

리눅스에서의  사용자((계정))
- super user : 리눅스  서버  시스템  전체를  관리할  수  있는  퍼미션을  갖는다.. 추가적으로  사용자,, 그룹등을  생성,, 삭제,, 수정할  수  있는  권한도  갖는다.. -> "root"
- normal user : 자신의  디렉토리((폴더))가  지정되고  해당  디렉토리에서는  모든  권한을  갖지만  다른  디렉토리에  대해서는  제한적인  권한을  갖는다..

- Linux -> Multi User, Multi Taking -> 다수의  사용자가  동시에  접속하여  자신만의  독립적인  작업  공간을  갖는다.. 

[root@localhost ~]# 
"root" -> 로그인한  사용자
"localhost" -> 현재  로그인한  서버의  이름
" ~ " -> 현재  로그인한  사용자가  있는  위치를  말하며  ~ 은  
           로그인한  사용자의  홈  디렉토리를  의미한다.. 

리눅스  파일  시스템은  아래와  같이  tree 구조를  갖는다.. 

                                  / (루트  파티션)
       --------------------------------------------------
       etc   var    home     usr      root     ...
        |                   |                    |
 sysconfig httpd...   user1  user2   Document Desktop... 

etc :  설치//설정  등을  위한  구성  파일이  위치한다
var :  주로  시스템  로그  등이  저장되는  디렉토리이다..
home : 일반사용자들의  홈  디렉토리

root :  super user 인  root 의  홈디렉토리
